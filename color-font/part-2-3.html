Opentype-SVG uses SVG markup language, a vector graphics file format that is widely used on the Web and provides rich graphic capabilities. The SVG description for a color glyph is a complete, integrated piece. No additional glyphs are required for the SVG description, which makes it beneficial for fonts that support a large number of characters. In comparison, in the COLR table, a color glyph required components that have separated glyph IDs.









Usually, Opentype-SVG is in a great position to become a golden standard in the four big players’ competition because there are three operating systems supporting it(Windows 10+, IOS 12+, macOS 10.14+). However, it usually has a larger file size compared to COLR and other color formats and fails to support variable glyph outlines. When making variable color fonts, Microsoft’s COLR/CPAL system is the only option until now, as the others are incompatible with the interpolation(variable outlines).
